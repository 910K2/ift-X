⚡ Breaking — Hikvision / Dahua banned from April 1, 2026
MeitY ER/STQC certification now mandatory for all internet-connected CCTV sold in India
Hikvision, Dahua, TP-Link and other Chinese brands cannot sell non-certified devices from April 1, 2026. Dahua has scaled back to analogue cameras only. Xiaomi and Realme have exited the smart camera segment entirely.
Market shift already happened: Indian brands now hold 80%+ of the market
In 2024, Chinese vendors held ~33% of the India CCTV market. Since the ER norms were introduced (April 2024), the market has restructured rapidly. Indian brands using Taiwanese chipsets and localised firmware are the new baseline.

What IFT's role actually is — honest definition
Your question is the right one: if MyGate provides the app and the API, what is IFT actually doing? The answer depends on which layer IFT chooses to own.
Three possible IFT positions — from weakest to strongest
Position A — Pure CCTV reseller + MyGate integrator (weakest)
IFT procures certified Indian CCTV hardware (CP Plus / Godrej), installs it, wires it to MyGate via the Hardware Access API. Resident sees MyGate app as before. IFT is invisible — just the installation contractor. Revenue model: one-time installation fee. No recurring revenue. No differentiation. Easily replaceable by any local electrician who can read an API doc. This is NOT a viable IFT position.
Position B — Managed integration layer with SLA (medium)
IFT installs and owns the physical-to-digital reliability layer. The CCTV feed, LPR accuracy, boom barrier response, and event notification latency are all under IFT's SLA — not MyGate's. MyGate provides the app; IFT guarantees that everything feeding into it works correctly and is diagnosable when it does not. Revenue model: monthly managed service fee (Rs 5,000–15,000/month per property depending on size). This is defensible because MyGate cannot own the physical layer and IFT's RCA capability makes the SLA credible. The risk: IFT is still dependent on MyGate's continued API availability and pricing decisions.
Position C — Full integration platform, MyGate as one output among several (strongest)
IFT builds its own cloud platform that aggregates CCTV events, access logs, LPR data, fuel sensor data, fleet GPS — and outputs to MyGate for residential communities, to hotel PMS for hotels, to fleet dashboards for logistics. MyGate is one API destination among several. IFT owns the data layer and the intelligence layer. Revenue model: platform subscription (per property or per device), plus RCA consulting retainer for complex deployments. This is InfoTrack X — the physical-to-digital integration platform that works across fleet, building, and hotel verticals. The risk: requires significant platform development investment before revenue.
The MyGate dependency question — brand and lock-in
If IFT integrates via MyGate API, who does the resident see?
The resident sees MyGate. The RWA committee sees MyGate. IFT is completely invisible to the end user. The gate opens, the notification arrives, the visitor is approved — all in the MyGate interface. IFT's work happens entirely in the background. This is a branding problem for IFT and a pricing leverage problem: if MyGate raises API fees or changes partner terms, IFT has no alternative.
MyGate's incentive to partner vs. compete
MyGate benefits from hardware partners because it does not want to manufacture or install physical devices. Its business model is software subscription, not hardware. So the partnership is structurally aligned — MyGate wants certified hardware integrators who can expand its installed base. However, MyGate could at any point certify a competing hardware integrator in the same property, or acquire one. IFT's protection is the quality of its physical integration and RCA capability — things that are hard to replicate quickly.
The incentive model that makes sense
IFT should not take revenue share from MyGate's subscription. IFT should charge separately for the physical integration and managed service layer. The RWA pays MyGate for the app (Rs 28,000–50,000/year). The RWA pays IFT separately for the physical integration and maintenance SLA (Rs 60,000–180,000/year depending on CCTV count and scope). These are independent contracts. IFT's value is not derived from MyGate's revenue — it is parallel to it.
Apartment internal communications — wired vs. wireless for CCTV
This is a real technical and commercial consideration for installation.
Current reality in Indian mid-size apartments
Most existing CCTV runs on wired coaxial or Cat5/6 — analogue DVR based
Older Hikvision installations are typically wired analogue. The DVR is on-site with no cloud connectivity. Footage is reviewed locally after incidents. No API. No real-time alerts. This is the replacement target.
Modern replacement: IP cameras over existing Cat5/6 or new PoE runs
IP cameras with PoE (Power over Ethernet) can reuse most existing cable infrastructure. NVR or cloud recorder connects via building Wi-Fi or dedicated LAN to internet. This is the standard upgrade path. Installation cost is manageable — no new conduit runs required if Cat5 is already in place.
Wireless (Wi-Fi) CCTV — viable for retrofits but has reliability limits
Wi-Fi cameras are easier to install in older buildings with no cable infrastructure. However, in dense apartment complexes with many residents, 2.4GHz / 5GHz interference, dead zones in basements and underground parking, and bandwidth contention are real problems. For security-grade reliability, wired PoE is preferable for critical entry points (main gate, lobby) and Wi-Fi acceptable for supplementary indoor coverage.
IFT's comms expertise is directly relevant here
The same problem IFT solves in fleet telematics — reliable data transmission from a device to a server, with diagnostics when it fails — applies directly to CCTV network reliability. Dropped frames, intermittent feeds, and storage failures in apartment CCTV are communication problems. Shaik's RCA capability is not just for fleet. It applies here too.
Summary — IFT's recommended position

Start with Position B (managed integration layer). Use MyGate API as the delivery mechanism for the first deployments — it reduces the sales barrier because RWAs already know and trust MyGate. IFT's pitch to the RWA is not "replace MyGate" but "make MyGate actually work reliably with your physical security infrastructure."

Build toward Position C over 12–18 months by developing IFT's own event aggregation platform that can output to MyGate, hotel PMS, and fleet dashboards. At that point IFT is no longer a MyGate integrator — it is a platform that happens to support MyGate as one output. The MyGate dependency dissolves without ever having to confront MyGate directly.

The brand question resolves itself through the SLA. The resident may not know IFT's name. But the RWA committee knows exactly who to call when the LPR camera stops reading plates at 2am. That is IFT's brand — not a logo on the app, but a name in the facilities manager's phone.
